# 智能手表项目需求规格说明书 (PRD)

## 1. 项目概览 (Project Overview)
本项目旨在开发一款基于 STM32 微控制器的可穿戴智能手表原型系统。该产品定位于嵌入式系统学习与验证平台，通过有限的硬件资源（Cortex-M3 + 20KB RAM），实现流畅的 UI 交互、精准的姿态解算以及多任务实时处理功能。

该项目不仅是一个功能演示，更是对**嵌入式软件分层架构**、**实时操作系统 (RTOS)** 以及**传感器数据融合算法**的深度实践。

## 2. 用户需求分析 (User Needs Analysis)
目标用户需要一款具备基础时间管理、健康监测及娱乐功能的便携式设备。

*   **核心诉求**：
    *   **操作响应迅速**：界面切换无卡顿，按键反馈及时。
    *   **交互直观**：菜单逻辑清晰，具备一定的智能化体验（如抬手亮屏）。
    *   **功能实用**：涵盖日常所需的计时、计步和简单的娱乐功能。
*   **使用场景**：日常佩戴查看时间、运动时记录步数、姿态检测演示、休闲娱乐。

## 3. 功能规格说明 (Functional Requirements)

### 3.1 核心功能模块
#### (1) 时钟系统
*   **实时显示**：支持年/月/日/时/分/秒的实时刷新显示。
*   **时间设置**：支持通过按键进入设置模式，对时间进行校准与调整。

#### (2) 运动健康
*   **计步器**：基于加速度传感器数据，通过波峰波谷检测算法统计步数，支持后台运行。
*   **姿态监测**：实时解算并显示 Pitch (俯仰)、Roll (横滚)、Yaw (偏航) 角度，具备电子水平仪功能。

#### (3) 实用工具
*   **秒表**：支持启动、暂停、复位功能，计时精度达到 10ms。
*   **手电筒**：控制板载 LED 进行照明，提供应急光源。

#### (4) 娱乐互动
*   **小恐龙游戏**：移植 Chrome 经典离线游戏，利用按键控制跳跃避障，具备碰撞检测与计分系统。
*   **表情包展示**：显示动态位图动画，增加趣味性。

### 3.2 交互体验设计
*   **物理交互**：三按键设计（上翻/增加、下翻/减小、确认/返回），逻辑简单易上手。
*   **体感交互**：**抬手亮屏**功能。通过 MPU6050 监测手腕姿态变化，当用户抬起手腕看表时自动唤醒屏幕。
*   **视觉体验**：菜单切换具备平滑的**滑动过渡动画**，拒绝生硬的页面跳转，提升高端感。

## 4. 非功能需求 (Non-Functional Requirements)
*   **实时性**：UI 刷新率需 > 30 FPS，按键响应延迟 < 50ms，保证操作跟手。
*   **稳定性**：在多任务并发（如后台计步 + 前台游戏）场景下，系统运行稳定，无死机、无堆栈溢出。
*   **可维护性**：代码需遵循企业级分层架构，驱动与应用分离，具备良好的可移植性和扩展性。
